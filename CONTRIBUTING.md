# 貢獻指南 / Contributing Guide

感謝你願意貢獻！這份文件說明如何新增產品、修改內容、以及提交 PR。

---

## 🚀 快速開始

### 方式一：直接在 GitHub 上改（最簡單）

1. 打開 [`data/products.csv`](data/products.csv)
2. 按 ✏️ 編輯按鈕
3. 在最後加上一行新產品
4. 提交 PR

### 方式二：Fork + Clone（適合大量修改）

```bash
git clone https://github.com/YOUR_USERNAME/iq-tax-products.git
cd iq-tax-products
# 編輯 data/products.csv
git add data/products.csv
git commit -m "Add: 新產品名稱"
git push
# 開 PR
```

## 📝 新增產品規範

### CSV 格式

每一行一個產品，欄位順序如下：

```
ID,分類,產品名稱,宣稱功效,打臉原因,違反原理,常見價格區間(台幣),關鍵字Tags,參考資料連結,備註,加入日期
```

### 欄位填寫說明

| 欄位 | 必填 | 說明 |
|------|------|------|
| ID | ✅ | 遞增流水號，取目前最大值 +1 |
| 分類 | ✅ | 四選一：`生物學` `物理學` `化學` `跨領域` |
| 產品名稱 | ✅ | 產品或技術名稱，不要加表情符號 |
| 宣稱功效 | ✅ | 廠商廣告詞，一句話摘要 |
| 打臉原因 | ✅ | 科學原理說明，讓國中生也看得懂 |
| 違反原理 | ✅ | 簡短版本，3-8 個字，如 `蛋白質消化分解` |
| 常見價格區間(台幣) | ✅ | 數字-數字，如 `500-3000`；無法估計填 `未知` |
| 關鍵字Tags | 建議 | 分號 `;` 分隔，3-5 個標籤，方便搜尋 |
| 參考資料連結 | 建議 | 論文 DOI / 新聞連結 / 政府公告。多個用分號分隔 |
| 備註 | 選填 | 任何補充說明或笑點 |
| 加入日期 | ✅ | YYYY-MM-DD 格式 |

### 範例

```csv
25,物理學,磁力瘦身腰帶,磁場燃脂甩掉小腹,脂肪沒有鐵磁性不受磁場影響,脂肪無鐵磁性,1000-5000,磁力;瘦身;腰帶;減肥,https://example.com/news/123,肚子的肉對磁鐵毫無反應,2026-06-15
```

## 🎯 挑選標準

### ✅ 符合收錄條件

- 產品宣稱與基礎科學原理矛盾（國中～高中程度即可判斷）
- 市面上真實存在且有人購買的消費產品
- 有明確可指出的科學錯誤

### ❌ 不收錄的

- 純粹主觀評價（「很難吃」「太貴了」）
- 個人直銷恩怨文
- 沒有科學根據的陰謀論（「政府隱瞞外星科技」類）
- 合法藥品/醫療器材的副作用討論（去藥局問藥師）

### 🤔 不確定的情況

- 開 Issue 討論，標題加 `[討論]` 前綴
- 傳統療法（中醫/阿育吠陀等）：只收明顯違反基礎科學的，不涉及醫學體系之爭

## 🔬 資料來源建議

### 好的來源
- 學術論文（PubMed / Google Scholar）
- 政府主管機關公告（FDA 裁罰紀錄 / 食藥署公告）
- 具公信力科普媒體（科學月刊、PanSci 泛科學）
- 消基會檢驗報告
- 廠商被裁罰的新聞報導

### 不夠好的來源
- 個人部落格（除非作者是該領域專家）
- YouTube 影片（除非有可靠實驗數據）
- 論壇討論串
- AI 生成的內容

## 📋 PR 檢查清單

提交前請確認：

- [ ] ID 沒有重複
- [ ] 分類是四個之一
- [ ] 打臉原因有科學依據，不是個人感覺
- [ ] 參考資料連結有效且權威
- [ ] 關鍵字 Tags 用分號分隔
- [ ] 加入日期是今天
- [ ] CSV 格式正確（用 Excel 或文字編輯器確認）

## 🏷️ Issue 標籤說明

| 標籤 | 用途 |
|------|------|
| `新產品建議` | 提議新增產品 |
| `資料勘誤` | 現有資料有誤 |
| `討論` | 科學爭議或分類討論 |
| `加強佐證` | 現有項目需要補參考資料 |
| `功能建議` | 對專案本身的建議 |
| `good first issue` | 新手友善任務 |

## 🤝 行為準則

- 理性討論，對事不對人
- 科學證據 > 個人經驗 > 網路傳說
- 這裡不是直銷檢舉版，是科學資料庫
- 可以嘴產品，不要人身攻擊
